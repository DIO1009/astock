## 2. 首次初始化

> 只需执行一次。

### 2.1 安装 Python 数据下载依赖

系统使用 AKShare 下载真实 A 股行情（不复权，免费，无需 API Key）：

```bash
pip3 install akshare
```

验证安装：

```bash
python3 -c "import akshare; print(akshare.__version__)"
```

### 2.2 下载真实行情数据（首次必须）

```bash
python3 scripts/fetch_data.py
```

脚本说明：

- 自动禁用本地 VPN 代理，直连东方财富/新浪财经数据源
- 下载最近 **120 个交易日**日线数据（不复权，与券商行情一致）
- 默认标的：600519（茅台）、000858（五粮液）、300750（宁德时代）、000300（沪深300）
- 输出文件：`real_market_data.csv`

```
======================================================

  日期范围: 20250901 → 20260323
  目标天数: 120 交易日
  输出文件: real_market_data.csv
======================================================
  [600519] 贵州茅台 ... 122 条
  [000858] 五粮液 ... 122 条
  [300750] 宁德时代 ... 122 条
  [000300] 沪深300 ... 122 条
======================================================
  ✅ 下载完成：共 488 条记录 → real_market_data.csv
```

如需自定义标的：

```bash
python3 scripts/fetch_data.py --symbols 600519 000858 300750 000300 601318 --days 120
```

### 2.3 确认 PostgreSQL 已启动

```bash
pg_isready -h 127.0.0.1 -p 5432 -U postgres
# 输出: 127.0.0.1:5432 - accepting connections
```

如果未启动：

```bash
brew services start postgresql@16
```

### 2.2 确认数据库存在

```bash
PGPASSWORD=dmrxlbol123 psql -h 127.0.0.1 -U postgres -lc | grep astock_trade
```

如果不存在则创建：

```bash
PGPASSWORD=dmrxlbol123 createdb -h 127.0.0.1 -U postgres astock_trade
```

> **数据表会在第一次启动时由程序自动创建，无需手动建表。**

## 3. 启动模式说明

系统提供三种运行模式，通过环境变量区分，**与"模拟盘/实盘"无关——无论哪种模式，持仓执行的都是纸面交易（Paper Trading），不会真实下单到券商**。区别在于策略使用的是历史价格还是实时价格。

### 模式对比总览

| 模式                 | 数据来源               | 价格类型              | Tick 间隔   | 适用场景                   |
| -------------------- | ---------------------- | --------------------- | ----------- | -------------------------- |
| **历史回放**（默认） | `real_market_data.csv` | 历史日线收盘价        | 1 秒/tick   | 策略验证、参数调优         |
| **实时行情**         | 东方财富 push2 API     | 实时最新价（Level-1） | 5 分钟/tick | 观察策略在实盘价格下的表现 |
| **自定义数据**       | 任意 CSV 文件          | 取决于文件内容        | 1 秒/tick   | 回测特定时段               |

---

### 模式一：历史回放模式（默认）

**不设置任何额外环境变量**，系统自动加载 `real_market_data.csv`，每秒推进一个历史交易日的收盘价数据。

```bash
# 标准启动，走历史回放
bash scripts/start.sh
```

**价格行为**：每个 tick 对应 CSV 里的一行历史记录。例如你在下午 6 点启动，系统会从 2025-07-22 的收盘价开始，每秒前进一个交易日，几分钟内就"跑完"了几个月的历史行情。

**注意**：

- 日志中看到的 `index=000300 price=4271→4288→4378` 是历史上不同日期的价格在快速播放，**不是当前市场价**
- 适合策略逻辑验证，**不能用于判断当前持仓是否盈亏**
- CSV 数据耗尽后，系统会持续使用最后一根 bar 的价格，直到手动停止

---

### 模式二：实时行情模式（接实盘数据）

设置环境变量 `ASTOCK_LIVE_DATA=1`，系统改用东方财富 push2 API 获取实时 Level-1 行情快照，不再读取 CSV。

```bash
# 启动实时行情模式
ASTOCK_LIVE_DATA=1 bash scripts/start.sh
```

或编辑 `scripts/start.sh`，在 Go 程序启动行前加入：

```bash
export ASTOCK_LIVE_DATA=1
```

**价格行为**：

- 每 5 分钟请求一次东方财富 API，获取 f43（最新价）、f60（昨收价）、买一/卖一等字段
- 价格和你在东方财富 App 或券商客户端看到的行情一致（Level-1，延迟约 3 秒）
- 策略信号、止损止盈触发都基于实时价格计算

**交易时间限制（重要）**：

> ⚠️ **当前版本引擎未内置交易时间校验**。非交易时段（收盘后、节假日）东方财富 API 返回的最新价为 0，系统会使用**缓存的上一次已知价格（通常为昨收价）**继续运行。
>
> 这意味着：
>
> - **非交易时段产生的信号和开单记录不反映真实市场意图**，仅供参考
> - **建议仅在交易时段内使用实时模式运行策略**：A 股交易时段为 **09:30–11:30 / 13:00–15:00**
> - 交易时段之外启动，应等到 09:30 开盘后再观察信号质量

**Tick 间隔覆盖**（可选）：

```bash
# 将 tick 间隔改为 1 分钟（默认实时模式为 5 分钟）
ASTOCK_LIVE_DATA=1 ASTOCK_TICK_SECONDS=60 bash scripts/start.sh
```

---

### 模式三：自定义数据文件

通过 `ASTOCK_DATA_PATH` 指定任意格式兼容的 CSV 文件，用于回测特定时间段或自定义标的。

```bash
# 指定自定义数据文件
ASTOCK_DATA_PATH=/path/to/my_data.csv bash scripts/start.sh
```

CSV 格式要求（必须包含表头）：

```
date,symbol,open,high,low,close,volume
2026-01-02,600519,1450.0,1480.0,1440.0,1465.0,4200000
...
```

### 三种模式的启动日志特征

启动后查看日志，可以通过以下关键字确认当前模式：

```bash
tail -20 logs/$(ls -t logs/ | head -1)
```

| 日志关键字                                  | 对应模式               |
| ------------------------------------------- | ---------------------- |
| `✅ 使用真实行情数据: real_market_data.csv` | 历史回放（默认）       |
| `✅ 实时行情模式（东方财富 push2 API）`     | 实时行情               |
| `✅ 使用指定数据文件 (ASTOCK_DATA_PATH)`    | 自定义数据             |
| `⚠️  未找到真实数据文件，生成合成数据`      | 兜底合成数据（应避免） |

## 6. 人工干预操作

系统提供三种实时干预方式：**前端按钮**、**Unix 信号**、**REST 命令**。

### 6.1 前端控制面板（推荐）

浏览器右侧"控制面板"区域：

| 按钮         | 效果                                   |
| ------------ | -------------------------------------- |
| **停止开仓** | 立即禁止所有新买入，已有持仓不受影响   |
| **恢复开仓** | 解除人工停止，恢复正常策略买入         |
| **全部清仓** | 需点击两次确认（5 秒内），强平所有持仓 |

### 6.2 命令行信号（终端操作）

```bash
# 先查询当前 PID
PID=$(cat scripts/pids)

# 停止新开仓（保留平仓）
kill -USR1 $PID

# 恢复开仓
kill -HUP $PID

# 强制全部清仓（谨慎）
kill -USR2 $PID
```

## 7. 日常停止

在项目根目录执行：

```bash
bash scripts/stop.sh
```

停止流程：

1. 发送 `SIGTERM`，程序优雅退出
2. **自动保存持仓快照** → `position_state.json`
3. 等待 3 秒，如未退出则强制 `SIGKILL`
4. 清理 PID 文件

```
▶ 正在停止交易后端 (PID=12345)...
  ✓ 已停止
============================================================
  服务已停止。持仓快照保存于: position_state.json
  查看日志: ls -lt logs/ | head -5
============================================================
```

> **重要：下次启动时会自动从 `position_state.json` 恢复持仓，防止重启丢失。**

## 8. 紧急停止

如果前端无法操作、信号无响应，使用强制终止：

```bash
# 方式 1：按进程名强杀
pkill -9 paper_trader

# 方式 2：按 PID 强杀
kill -9 $(cat scripts/pids) 2>/dev/null; rm -f scripts/pids
```

> ⚠️ 强制终止不会保存持仓快照，下次启动将从空仓开始。

## 9. 数据查询

### 9.1 REST API（浏览器 / curl）

```bash
# 资金曲线（最近 7 天）
curl "http://localhost:18099/api/equity?range=7d"

# 资金曲线（全部）
curl "http://localhost:18099/api/equity?range=all"

# 最近 50 笔成交记录
curl "http://localhost:18099/api/executions?limit=50"

# 指定股票成交记录
curl "http://localhost:18099/api/executions?symbol=600519&limit=20"

# 当前持仓
curl "http://localhost:18099/api/positions"

# 风控事件
curl "http://localhost:18099/api/risk-events?limit=20"

# 最新系统状态
curl "http://localhost:18099/api/system-status/latest"
```

### 9.2 直接查询 PostgreSQL

```bash
# 连接数据库
PGPASSWORD=dmrxlbol123 psql -h 127.0.0.1 -U postgres -d astock_trade
```

常用查询：

```sql
-- 最近 20 笔成交
SELECT symbol, side, qty, price, status, latency_ms,
       to_timestamp(execution_time/1000) AS exec_at
FROM executions
ORDER BY execution_time DESC
LIMIT 20;

-- 资金曲线（最近 50 个点）
SELECT to_timestamp(timestamp/1000) AS ts,
       round(equity::numeric, 2)    AS equity,
       round(drawdown::numeric, 4)  AS dd_pct
FROM equity_curve
ORDER BY timestamp DESC
LIMIT 50;

-- 当前持仓
SELECT symbol, qty, avg_price,
       round(market_value::numeric, 2)   AS market_val,
       round(unrealized_pnl::numeric, 2) AS pnl_pct
FROM positions;

-- 所有风控事件
SELECT to_timestamp(timestamp/1000) AS ts,
       event_type, round(drawdown::numeric,2) AS dd_pct, description
FROM risk_events
ORDER BY timestamp DESC;

-- 最新系统状态
SELECT to_timestamp(timestamp/1000) AS ts,
       streak, risk_level, is_opening_allowed, is_kill_switch_active
FROM system_status
ORDER BY timestamp DESC
LIMIT 1;
```

## 11. 常见问题

### Q1：启动后浏览器显示"重连中…"

**原因**：后端进程未启动或端口被占用。

```bash
# 检查进程
cat scripts/pids && kill -0 $(cat scripts/pids) && echo "运行中"

# 检查端口
lsof -i :18099

# 查看启动日志
tail -50 logs/$(ls -t logs/ | head -1)
```

---

### Q2：提示"DB 连接失败"但程序继续运行

**原因**：PostgreSQL 未启动或密码错误，系统自动降级为纯文件模式。

```bash
# 检查 PostgreSQL
pg_isready -h 127.0.0.1 -p 5432

# 手动测试连接
PGPASSWORD=dmrxlbol123 psql -h 127.0.0.1 -U postgres -d astock_trade -c "SELECT 1"
```

---

### Q3：Mac 休眠后数据停止更新

**原因**：`caffeinate -sim` 仅在**接 AC 电源**时阻止休眠。

**解决方案**：

- 确保 Mac 连接电源适配器
- 或临时设置系统永不休眠：
  ```bash
  sudo pmset -a sleep 0 disksleep 0 displaysleep 0
  # 恢复默认：sudo pmset -a sleep 1
  ```

---

### Q4：前端"全部清仓"按钮无反应

**原因**：需要在 5 秒内**点击两次**确认（防误操作）。

**备用方案（命令行）**：

```bash
kill -USR2 $(cat scripts/pids)
```

---

### Q5：想查看某支股票的历史成交明细

```bash
curl "http://localhost:18099/api/executions?symbol=600519&limit=100" | python3 -m json.tool
```

---

### Q6：系统提示连续亏损冻结开仓，如何手动解冻

等待冻结 tick 结束（默认 50 tick），或人工恢复：

```bash
# 命令行
kill -HUP $(cat scripts/pids)

# 前端：点击"恢复开仓"按钮
```

## 附录：价格体系说明

系统内部统一使用**不复权实时收盘价**，规范如下：

| 价格类型        | 来源                          | 说明                             |
| --------------- | ----------------------------- | -------------------------------- |
| Market Price    | AKShare `adjust=""`           | 不复权日线收盘价，与券商行情一致 |
| Execution Price | Market Price × (1 ± slippage) | 在市价基础上加滑点模型           |
| Signal Price    | Market Price                  | 策略判断使用同一价格源           |
| Display Price   | Market Price                  | 前端展示价格                     |

**价格一致性验证**：

```bash
# 查看系统最新成交价格
curl "http://localhost:18099/api/executions?limit=5" | python3 -m json.tool

# 对比 real_market_data.csv 中最新收盘价
tail -10 real_market_data.csv | grep 600519
```

如果发现价格偏差 > 1%，排查步骤：

1. 确认 `real_market_data.csv` 存在且数据新鲜（查看文件日期）
2. 重新下载：`python3 scripts/fetch_data.py`
3. 重启系统：`bash scripts/stop.sh && bash scripts/start.sh`
4. 查看启动日志是否显示 `✅ 使用真实行情数据: real_market_data.csv`
